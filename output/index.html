The "don't make me think" version of how to release and maintain FOSS Python code: low on choices, long on opinions.

Lots of ways to do it; this way works.

all the things

  • project structure
  • choosing a license
  • code hosting
  • documentation
  • testing & CI
  • packaging
  • being a good maintainer

assumptions

  • not huge
  • one developer or small team
  • target: pip install

If you're releasing OpenStack and/or have a team of four hundred developers, things look a bit different. Much more custom needs.

Depending what you're building, many ways to get your code to users. This talk will assume pip install; good for libraries or simpler applications. Won't be talking about e.g. Windows or OS X installers, Linux distro packaging, web application deployment
